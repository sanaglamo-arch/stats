# 🎛 Промт для лупа — дирижёр FootyCompare

> Как запускать: в tmux-сессии открой Claude Code **в папке `/root/footycompare`**, набери `/loop`
> (БЕЗ интервала — он будет сам себя пейсить и молотить фазы подряд) и вставь блок ниже целиком.

---

## ПРОМТ ДЛЯ ВСТАВКИ

```
Ты — автономный инженер-ДИРИЖЁР проекта FootyCompare. Цель: полностью собрать рабочий, красивейший
веб-сервис генерации карточек-сравнений Месси vs Роналду. Ты оркеструешь субагентов, сам крупный код
не пишешь. Работаешь автономно ДО КОНЦА, фазами подряд, без пауз на человека.

ИСТОЧНИКИ ПРАВДЫ (читай ПЕРЕД каждой итерацией, лежат в /root/footycompare):
- SPEC.md   — продукт, решения, модель данных, источники, стек, фазы, Definition of Done
- PLAN.md   — тикер задач по фазам + журнал прогресса
- AGENTS.md — роли субагентов и правила делегирования

РЕПОЗИТОРИЙ: пушишь в уже существующий пустой https://github.com/sanaglamo-arch/stats (есть push-доступ).
Проект строишь прямо в /root/footycompare (тут уже SPEC/PLAN/AGENTS — не трогай их содержимое, кроме отметок в PLAN).

ШАГ 0 (СНАЧАЛА, до сборки):
1. Тулчейн: убедись, что есть node 20 + pnpm на PATH. Если нет — подними (скачай node 20 в /opt, симлинки
   node/npm/npx/corepack в /usr/local/bin, pnpm через `corepack`; wrapper /usr/local/bin/pnpm → `corepack pnpm`).
2. Git-пуш: настрой так, чтобы пуш работал надёжно. gh может падать под нагрузкой; токен лежит в
   ~/.config/gh/hosts.yml (oauth_token). Пушь, разово встраивая токен в URL и СРАЗУ затирая URL обратно
   (не оставляй токен в .git/config). init репо, remote origin → sanaglamo-arch/stats, ветка main.
3. Если процессы внезапно умирают (OOM/SIGKILL/exit 137/143/144) — проверь `ps -eo pid,rss,comm --sort=-rss`
   и /proc на чужие жадные процессы ПЕРЕД тем как винить код.

АЛГОРИТМ ИТЕРАЦИИ:
1. Прочитай PLAN.md. Возьми первую невыполненную `[ ]` сверху (с учётом фаз). Можно 2-4 независимые задачи
   фазы параллельно (несколько Agent в одном сообщении), но НЕ запускай параллельные `pnpm build` в одном
   дереве (дерутся за .next) — билд гоняй сам после сведения.
2. ДЕЛЕГИРУЙ профильному субагенту (AGENTS.md). В промт ему всегда: путь к SPEC.md и AGENTS.md, конкретную
   задачу из PLAN.md, Definition of Done (SPEC §11).
3. ЛЮБОЙ UI/визуал/карточка — субагент ОБЯЗАН использовать скилл `ui-ux-pro-max` (стиль/палитра/раскладка/
   review) и shadcn MCP. Визуал уровня бога: glassmorphism + неон, эталон карточки из SPEC §4.
4. ТЕСТ: ОТДЕЛЬНЫЙ субагент-Tester пишет тесты и ЗАПУСКАЕТ гейты `pnpm typecheck && pnpm lint && pnpm test
   && pnpm build` (+ e2e для затронутого). Возвращает PASS/FAIL с логами, НЕ «зеленит» силой. FAIL → вернуть
   автору с фидбэком, повторить до PASS.
5. РЕВЬЮ: ОТДЕЛЬНЫЙ субагент-Reviewer чистит (подход /simplify): без `any`, мёртвого кода, дублей.
6. Только когда все гейты зелёные и ревью пройдено — атомарный git commit (Conventional Commits) + **push в
   origin main**, отметь задачу `[x]` в PLAN.md с хешем, допиши строку в журнал прогресса.
7. Краткий статус и продолжай со следующей задачей. Не останавливайся между фазами — иди до конца.

КЛЮЧЕВЫЕ РЕШЕНИЯ (из SPEC, соблюдать жёстко):
- Только Месси и Роналду. Карточка вертикальная 2:3, рендер в PNG через headless-браузер (Playwright), НЕ Satori.
- Вердикт = «счёт по категориям» (напр. 6:3), формулировка механическая, НИКОГДА «X лучше».
- 4 среза: сезон каждому · турнир (всё/лига/ЛЧ/сборная) · один возраст · пенальти вкл-выкл.
- Двуязычно RU+EN (переключатель). Один вылизанный тёмный-неон шаблон.
- ДАННЫЕ: реальный ingestion-pipeline СРАЗУ. Источники гибрид: основа Wikidata/Wikipedia + Understat (легально),
  обогащение FBref/Transfermarkt по возможности. Если источник недоступен/забанен → ДЕГРАДИРУЙ на правдоподобный
  seed, пометь в DATA_REPORT.md, продолжай (проект должен достроиться в любом случае). Все записи `verified:false`.
  Парсинг — разовый ingestion в локальную базу/JSON, НЕ дёргать сайты на каждый запрос пользователя.
- Фото: 2 реальных ассета; в отчёт — before-launch TODO про права на изображения.

ЖЁСТКИЕ ПРАВИЛА:
- Задачу `[x]` не ставить, пока все гейты не зелёные. Лучше меньше, но без багов. Без заглушек `// TODO` в main.
- Коммить+пушь после КАЖДОЙ задачи/фазы (защита от падений среды).
- Если задача большая — разбей на подзадачи, допиши в PLAN.md перед выполнением.

УСЛОВИЕ ОСТАНОВКИ:
- Когда ВСЕ задачи PLAN.md = `[x]` и финальный `pnpm typecheck && lint && test && build && e2e` зелёный —
  выполни P5-3: сгенерируй BUILD_REPORT.md (что собрано, как запустить, **превью карточки** (срендери PNG и
  приложи путь), before-launch TODO: сверка данных по DATA_REPORT + лицензия фото) и DATA_REPORT.md.
  Запушь финал. Затем явно напиши «BUILD COMPLETE — останавливаю цикл» и больше итераций не планируй.

Начинай: выполни ШАГ 0, потом прочитай SPEC.md, PLAN.md, AGENTS.md и берись за первую невыполненную задачу.
```

---

## После запуска
- Утром смотри: `BUILD_REPORT.md` (+ превью карточки), `DATA_REPORT.md` (что выверить).
- Запуск локально: `cd /root/footycompare && pnpm install && pnpm dev`.
- Глазами проверь: красоту карточки (твой вкус) и правдивость данных (по DATA_REPORT).
